# Kepi Search

Personal **Hyatt / Marriott / Hilton** hotel explorer: pick a city, draw a rectangle on the map, and search the catalog inside that area. Results include **walking-time estimates** (OpenRouteService when configured, otherwise the public OSRM foot demo), **chain deep links**, and filters for chains and sort order.

## Quick start

```bash
npm install
cp .env.example .env.local
# Add NEXT_PUBLIC_MAPTILER_KEY from https://www.maptiler.com/
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Use `?city=venice` (or any catalog id) in the URL.

## Environment

| Variable | Required | Purpose |
|----------|----------|---------|
| `NEXT_PUBLIC_MAPTILER_KEY` | **Yes** for the map | MapLibre style tiles (MapTiler Streets) |
| `OPENROUTESERVICE_API_KEY` | No | Foot routing via OpenRouteService instead of public OSRM |

## Session persistence and share links

- After a successful search, the app saves the **city**, **search polygon**, **results**, and **filter state** to `localStorage` (`kepi-search-session-v1`). Reloading the same city restores the rectangle and list when possible.
- Successful searches also update the URL with `city` and a compact JSON `area` parameter when the polygon is small enough.
- **Copy link** copies the current city + drawn rectangle to the clipboard so you can share a reproducible view. If the polygon is too large for the URL, the link falls back to city only and a hint is shown.

## City data

- **Bulk seeds** live in `src/data/city-seeds.json`, generated by `npm run seed:generate` from `scripts/generate-city-seeds.mjs` (plus `scripts/data/us-city-table.json` for United States rows).
- **Rich overrides** live in `src/data/cities/*.ts` and replace seeds for the same `id` in `src/data/cities/registry.ts` (e.g. Venice, London, **New York**, **Los Angeles**, **Chicago** with real property rows).
- To add hotels for another city: either extend `city-seeds.json` via the generator, or add a `CityCatalog` file and register it in `registry.ts`.

### OSM import helper (optional)

```bash
node scripts/fetch-hotels-overpass.mjs 40.74 -74.00 40.77 -73.96
```

Prints JSON for hotels whose `name` matches Marriott / Hilton / Hyatt heuristics inside the bbox. **Review and de-duplicate** before pasting into seeds or TS catalogs. Be kind to the [Overpass API](https://operations.osmfoundation.org/policies/overpass/).

## Deploy (e.g. Vercel)

1. Push the repo and import it in [Vercel](https://vercel.com/).
2. Set `NEXT_PUBLIC_MAPTILER_KEY` (and optionally `OPENROUTESERVICE_API_KEY`) in Project → Settings → Environment Variables for **Production** (and Preview if you use previews).
3. Default build: `npm run build`, output Next.js app. No extra server beyond API routes.

**Branch:** This repo’s active line of work lives on **`master`**. GitHub’s default branch may still be **`main`**. In Vercel → *Settings → Git → Production Branch*, point production at **`master`**, or keep **`main`** fast-forwarded to the same commit as `master`—otherwise Vercel keeps building an old tree while you push to `master`.

## Stack

Next.js (App Router), TypeScript, MapLibre GL, MapTiler, Terra Draw, Turf, Zod. Search runs on the server in `src/app/api/search/route.ts`.
